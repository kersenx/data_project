# Data Project

## 每日股市变化梳理

本仓库包含一个 GitHub Actions 定时任务，用于在每日 16:00（Asia/Shanghai）生成股市变化 Markdown 报告。

- 定时任务文件：`.github/workflows/daily-market-summary.yml`
- 生成脚本：`scripts/daily_market_summary.py`
- 默认输出目录：`reports/`

工作流会在 08:00 UTC 触发，对应 Asia/Shanghai 的 16:00。也可以在 GitHub Actions 页面通过 `workflow_dispatch` 手动触发。

### 自定义关注标的

默认脚本会梳理上证指数、深证成指、创业板指、恒生指数、日经 225、标普 500、纳斯达克和道琼斯。若需调整范围，可以在工作流中设置 `MARKET_TICKERS`：

```bash
MARKET_TICKERS='上证指数=000001.SS,恒生指数=^HSI,标普500=^GSPC'
```

格式为以英文逗号分隔的 `名称=Yahoo Finance代码`。

### 本地运行

```bash
pip install -r requirements.txt
python scripts/daily_market_summary.py
```
# 数据科学与数据开发文档库

本仓库用于创建、整理和维护数据科学与数据开发相关的文档信息，帮助团队沉淀知识、统一规范，并提升数据项目的协作效率。

## 目标

- 汇总数据科学、数据分析、数据工程和数据开发相关资料。
- 记录常用流程、最佳实践、技术规范和项目经验。
- 为数据项目提供可复用的文档模板和参考指南。
- 促进团队成员之间的知识共享与协作。

## 建议内容

本仓库可用于维护以下类型的文档：

- 数据科学方法论与建模流程
- 数据开发规范与工程实践
- 数据仓库、数据湖和数据平台相关说明
- ETL/ELT 流程设计与任务说明
- 数据质量、数据治理和元数据管理文档
- 指标体系、数据字典和业务口径说明
- 常用工具、脚本和环境配置说明
- 项目复盘、问题排查和经验总结

## 目录建议

后续可根据实际需要逐步扩展目录结构，例如：

```text
.
├── docs/              # 通用文档与指南
├── templates/         # 文档模板
├── standards/         # 规范与标准
├── examples/          # 示例与案例
└── README.md          # 仓库说明
```

## 使用方式

1. 根据主题在合适目录下新增或更新文档。
2. 文档命名应尽量清晰，便于检索和维护。
3. 对重要概念、数据口径和流程变更进行版本记录。
4. 提交前检查格式、链接和内容准确性。

## 贡献说明

欢迎补充和完善数据科学/数据开发相关文档。建议提交内容时包含：

- 文档目的和适用范围
- 关键背景信息
- 具体步骤或规范说明
- 示例、注意事项或参考链接
